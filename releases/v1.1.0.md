# GraphQL Cascade v1.1.0 Release Notes

**Release Date**: 2025-12-04
**Type**: Minor version (backward compatible)

## Overview

Version 1.1 adds production-ready error codes for common operational scenarios while maintaining 100% backward compatibility with v1.0.

## What's New

### New Error Codes

Three new error codes for production scenarios:

- **`TIMEOUT`**: Operation exceeded time limit (database, external service)
- **`RATE_LIMITED`**: Client exceeded rate limit
- **`SERVICE_UNAVAILABLE`**: Upstream dependency unavailable

### Enhanced Documentation

- **Error Code Selection Guidelines**: Comprehensive guide for choosing appropriate error codes
- **Async Operation Patterns**: Documented recommended patterns for asynchronous mutations
- **Extended Examples**: Real-world examples for all error codes

## Backward Compatibility

âœ… **Fully backward compatible** - All changes are additive:

**No migration required** - v1.1 is opt-in:

- Continue using v1.0 error codes indefinitely
- Adopt new codes incrementally as needed
- Mixed v1.0/v1.1 usage supported

See [Migration Guide](../docs/migration/v1.0-to-v1.1.md) for details.

## Use Cases

### Timeout Handling

```json
{
  "errors": [{
    "message": "Database query timed out",
    "code": "TIMEOUT",
    "extensions": {
      "timeoutMs": 5000,
      "retryable": true
    }
  }]
}
```

### Rate Limiting

```json
{
  "errors": [{
    "message": "Rate limit exceeded",
    "code": "RATE_LIMITED",
    "extensions": {
      "retryAfter": 60,
      "limit": 100,
      "window": "1m"
    }
  }]
}
```

### Service Degradation

```json
{
  "errors": [{
    "message": "Email service unavailable",
    "code": "SERVICE_UNAVAILABLE",
    "extensions": {
      "service": "email-provider",
      "retryable": true
    }
  }]
}
```

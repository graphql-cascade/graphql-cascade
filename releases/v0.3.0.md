# GraphQL Cascade v0.3.0 - Production Readiness

**Release Date**: December 5, 2025
**Previous Version**: v0.2.0 (unreleased)
**Status**: Stable for production use

## Overview

GraphQL Cascade v0.3.0 marks the first production-ready release with comprehensive error handling and improved developer experience. This release focuses on stability, observability, and production hardening while maintaining full backward compatibility.

## What's New

### Enhanced Error Handling (Spec v1.1 Compliance)

Complete implementation of the Cascade specification v1.1 error codes:

- **Network Errors**: `TIMEOUT`, `RATE_LIMITED`, `SERVICE_UNAVAILABLE`
- **Client Errors**: `VALIDATION_ERROR`, `NOT_FOUND`, `UNAUTHORIZED`, `FORBIDDEN`
- **Server Errors**: `CONFLICT`, `TRANSACTION_FAILED`, `INTERNAL_ERROR`

### Developer Experience Improvements

- **Error Utilities**: Convenience functions for error creation and handling across all client packages
- **Type Safety**: Enhanced TypeScript types with `CascadeErrorInfo` (renamed from `CascadeError`)
- **Comprehensive Tests**: Full test coverage for error scenarios
- **Version Alignment**: Consistent versioning strategy across all packages

## Migration Guide

### From v0.1.0

```typescript
// Before (v0.1.0)
import { CascadeError } from '@graphql-cascade/client';

// After (v0.3.0)
import { CascadeErrorInfo } from '@graphql-cascade/client';
```

### Error Handling

```typescript
// Enhanced error handling with new error codes
import { createCascadeError, isCascadeError } from '@graphql-cascade/client';

try {
  // Your GraphQL operation
} catch (error) {
  if (isCascadeError(error)) {
    switch (error.code) {
      case 'TIMEOUT':
        // Handle timeout
        break;
      case 'RATE_LIMITED':
        // Handle rate limiting
        break;
      case 'VALIDATION_ERROR':
        // Handle validation errors
        break;
      // ... handle other error codes
    }
  }
}
```
